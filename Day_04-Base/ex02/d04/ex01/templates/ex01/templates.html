Ex01 : Moteur de template.

Le moteur de templates Django

Le moteur de templates Django permet de générer du HTML dynamiquement. Il sépare la présentation (HTML) de la logique métier (Python).

1. Les blocs ({% verbatim %}{% block %}{% endverbatim %})

Les blocs permettent l'héritage de templates. Un template enfant peut redéfinir le contenu d'un bloc défini dans le template parent. 

{% verbatim %}



    {% block title %}Titre par défaut{% endblock %}


    {% block content %}{% endblock %}




{% extends 'base.html' %}
{% block title %}Mon titre{% endblock %}
{% block content %}
Mon contenu
{% endblock %}
{% endverbatim %}

2. Les boucles for...in

Les boucles {% verbatim %}{% for %}{% endverbatim %} permettent d'itérer sur des listes, tuples, ou dictionnaires. 

{% verbatim %}
{% for item in items %}
    
  • {{ item }}
    • 
{% endfor %}
{% endverbatim %}

    Exemple avec les données du contexte :

      {% for item in items %}
    • {{ item }}
      • {% endfor %}

    3. Les structures if

    Les conditions {% verbatim %}{% if %}{% endverbatim %} permettent d'afficher du contenu conditionnel. 

    {% verbatim %}
{% if show_advanced %}
    
    Contenu avancé
    
{% else %}
    
    Contenu basique
    
{% endif %}
{% endverbatim %}

    Résultat :

    {% if show_advanced %}

    ✅ Le contenu avancé est affiché car show_advanced = True

    {% else %}

    Contenu basique

    {% endif %}

    4. Affichage de variables (contexte)

    Les variables passées par le contexte (depuis la vue) sont affichées avec {% verbatim %}{{ variable }}{% endverbatim %}. 

    {% verbatim %}
# views.py
context = {'nom': 'Django', 'version': '6.0'}
return render(request, 'page.html', context)

# page.html

    Framework: {{ nom }} {{ version }}
    
{% endverbatim %}
