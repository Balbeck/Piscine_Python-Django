Ex01 : Processus d'affichage d'une page statique.

1. Requête HTTP

L'utilisateur entre une URL dans son navigateur (par exemple : http://127.0.0.1:8000/ex01/django/). Le navigateur envoie une requête HTTP GET au serveur Django.

2. Routage des URLs (urls.py)

Django reçoit la requête et consulte le fichier urls.py principal du projet. Il cherche un pattern d'URL correspondant à /ex01/django/. 

{% verbatim %}
# d04/urls.py
path('ex01/', include('ex01.urls'))

# ex01/urls.py
path('django/', views.django_view, name='django')
{% endverbatim %}

3. Exécution de la vue (views.py)

Django appelle la fonction django_view(request) définie dans views.py. Cette fonction contient la logique métier. 

def django_view(request):
    return render(request, 'ex01/django.html')

4. Chargement du template

La fonction render() charge le fichier template ex01/templates/ex01/django.html. Django parcourt tous les dossiers templates/ des applications installées dans INSTALLED_APPS.

5. Rendu du template

Django interprète le template HTML, remplace les variables du contexte (si présentes), exécute les tags de template ({% verbatim %}{% %}{% endverbatim %}), et génère le HTML final.

6. Réponse HTTP

Django envoie une réponse HTTP 200 avec le contenu HTML généré au navigateur. Le navigateur affiche la page à l'utilisateur.

Schéma récapitulatif

Navigateur → Requête HTTP
           ↓
       urls.py (routage)
           ↓
       views.py (logique)
           ↓
    Template HTML (rendu)
           ↓
       Réponse HTTP
           ↓
     Affichage dans le navigateur
